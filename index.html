Introduction

Git is a popular version control system. It is used for tracking code changes, tracking who made those changes, and allowing coding collaboration.

What you can do with Git:

  • Manage project with Repositories
  • Clone a project to work on a local copy
  • Control and track changes with Staging and Commiting
  • Branch and Merge to allow for work on different parts and versions of a project
  • Pull the latest version of the project to a local copy
  • Push local updates to the main project

Getting Started With Git

Configure Git

Let Git know who you are by entering a username and email. This will be helpful when viewing version history.

Initialize Git

Once you have created a folder for your project and navigated to it, initialize Git on that folder with git init

Clone a GitHub Repo

Navigate to your desired directory in your terminal and type: git clone git@github.com:<tagName>/<repoName>.git

Staging Environment

Git Status

Once you start editing and adding files to your project, you can check on the Git status to see if it is part of our repo

Files in the Git repo folder can be in one of 2 states:

  1. Tracked: files that Git knows about and are added to the repo
  2. Untracked: files that are in your working directory, but not yet added to the repo

Check the Git status with git status

Staging Files

Stages files are files that are ready to be committed to the repository you are working on

Add a file to the Staging Environment with git add <fileName>

  • Shortcut: use git add . to stage all changes

Git Commits

Adding commits keep track of our progress and changes as we work. Git considers each commit change point or "save point". It is a point in the project you can go back to if you find a bug, or want to make a change.

To add a commit with a message, do git commit -m "<message>"

Tip: You can view the history of commits for a repository with git log --oneline

Branches

A branch is a new/separate version of the main repository

Create a New Branch

git checkout -b <nameOfBranch>

Moving Between Branches

git checkout <nameOfBranch>

Tip: the shortcut git co -b <nameOfBranch> creates a branch and moves to it

Merge Branches

git merge <branchName>

Delete Branches

git branch -d <branchName>
Pulling/Pushing to GitHub

Run git fetch and git pull every time you enter into your project to get the most up to date version from dev

Tip: If you have been working on a branch for more than a day, run git merge dev on your working branch before you raise a PR. Adjust any merge conflicts as needed.

Use git push to push your changes to your branch after you have commited them

Help! Git Restore/Reset/Revert

Git Restore

Restore is about restoring files in the working tree from either the index or another commit. This command does not update your branch. The command can also be used to restore files in the index from another commit.

Use git restore --staged <file> to remove accidentally staged files.

Git Reset

Reset is about updating your branch, moving the tip in order to add or remove commits from the branch. This operation changes the commit history.

Use git reset #commitHash to remove commits from a branch.

Git Revert

Revert is about making a new commit that reverts the changes made by other commits.

Use git revert
